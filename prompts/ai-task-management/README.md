# 自然言語で動くAIタスク管理システム

Claude.ai / ChatGPT 等の既存チャットUIに `tasks-operate.md`（振る舞い定義）と `tasks.md`（タスクデータ）を貼り付けるだけで動く、プロンプト駆動のタスク管理システム。API キー・サーバー・カスタムアプリ不要。詳細は [requirements.md](requirements.md) 参照。

## アーキテクチャ

**2ファイル構成**。`tasks-operate.md` がAIへの振る舞い定義（システムプロンプト相当）、`tasks.md` がタスクデータの唯一の真実源。セッション開始時に両ファイルをコピペしてAIに渡し、自然言語で指示するだけで操作できる。

```
セッション開始:  tasks-operate.md + tasks.md
                → AI → 自動初期化（Inbox処理・トリアージ）の変更サマリー

セッション中:   自然言語の指示
                → AI → 変更サマリー（tasks.md 全体は返さない）

出力指示時:     「出力して」または「終了」
                → AI → tasks.md 全体（コードブロック）
                → コピー → tasks.md に上書き保存
```

## ファイル構成

| ファイル | 役割 |
|---------|------|
| `tasks-operate.md` | AIへの振る舞い定義。セッション開始時に tasks.md と一緒に貼り付ける |
| `tasks.md` | タスクデータのテンプレート。コピーして自分のタスクを管理する |
| `requirements.md` | 要件定義書 |

`tasks-operate.md` は**全文そのまま AI に貼り付けて使える**プロンプト本文のみを記載している。

## 前提環境

- Claude.ai / ChatGPT 等の対話型AIチャットUI（アカウントのみ必要）
- テキストエディタ（`tasks.md` の管理用）

## 運用フロー

### セッション開始

```
1. tasks-operate.md と tasks.md をテキストエディタで開く
2. 両ファイルの内容をコピーし、チャットUIに貼り付ける
3. 送信する
4. AIが自動でInbox処理・トリアージを実行し、変更サマリーを返す
```

### セッション中（何度でも）

```
1. 自然言語で指示を入力する
2. AIが変更サマリーを返す
```

指示の例:

| 指示例 | AIの動作 |
|--------|---------|
| 「資料作成が終わった」 | 該当タスクを `done` に更新 |
| 「週次レビューをして」 | 完了集計・保留見直し・翌週MIT候補を提示 |
| 「アーカイブして」 | `done` タスクを抽出しアーカイブ用内容を生成 |
| 「資料作成の締め切りを来週月曜に変えて」 | 該当タスクの `due` を更新 |

### セッション終了・tasks.md の保存

```
1. 「出力して」または「終了」と入力する
2. AIが tasks.md 全体をコードブロックで返す
3. コードブロックをコピーし、tasks.md に上書き保存する
```

### 新しいタスクのキャプチャ（セッション外）

AIを使わずに `tasks.md` の `## Inbox` セクションにメモを書いておくだけでよい。次のセッション開始時にAIが自動処理する。

## tasks.md のセクション構造

```
## Inbox          思いついたことを自由に書く（形式不問）
## MIT            最重要タスク（最大3件、AIが自動選定）
## Active         進行中・予定タスクの一覧表（テーブル形式）
## Someday/Maybe  いつかやるかもリスト
## Gantt Chart    ガントチャート（AIが自動更新）
## Dependency Graph  依存関係グラフ（AIが自動更新）
## Priority Matrix   優先度マトリクス（AIが自動更新）
```

ファイルは200行以内を目安に管理し、超過時は「アーカイブして」と指示する。アーカイブ先は `tasks-archive-YYYY-MM.md`（月単位）。
